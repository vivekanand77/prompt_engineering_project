BEFORE YOU BEGIN — WHAT YOU MUST KNOW
================================================================================

THE ONE-LINER PITCH:
"We built an interactive platform that teaches anyone how to talk to AI
 properly — and saves them money doing it."

THE PROBLEM YOU'RE SOLVING:
Billions of AI API calls happen every day. Most people write prompts by
trial-and-error. They waste tokens, get bad outputs, and have no idea why.
There's no single platform that combines LEARNING + BUILDING + TESTING.
Blog posts don't teach by doing. ChatGPT doesn't teach at all.

YOUR DIFFERENTIATOR:
This is NOT a chatbot. This is NOT a documentation site. This is an
interactive LAB where users learn through hands-on exercises, build prompts
with a structured tool, and test them against real AI models — all in one
place. Think of it as "Duolingo meets VS Code, but for AI prompts."

KEY NUMBERS TO MEMORIZE:
  - 10 interactive learning modules with auto-graded exercises
  - 6 battle-tested prompt templates used by real AI engineers
  - 4 AI models supported (GPT-4, Claude 3, Gemini Pro, Local LLM)
  - Side-by-side model comparison in one click
  - Full prompt management with export/import
  - Works in both light and dark mode
  - Fully responsive — works on phone, tablet, desktop

TECH STACK (if asked):
  Next.js 16, React 19, TypeScript, Framer Motion, Tailwind CSS, localStorage

SECTION 3: SIMULATION / JOURNEY — THE WOW FACTOR (90 seconds)
================================================================================

*** THIS IS YOUR SHOWSTOPPER. SPEND THE MOST TIME HERE. ***

WHAT TO SAY:
"This is the feature that makes us different from everything else out there.
 Most platforms tell you WHAT a token is. We SHOW you."

[Stage 1 — Live Tokenizer]

"Type any text and watch it get broken down into tokens in real-time.
 See how 'unbelievable' becomes ['un', 'believ', 'able']. Every colored
 block is one token — and every token costs money."

[Type something like "Artificial intelligence is transforming the world"]

"Notice how common words like 'the' are one token, but technical terms
 get split into pieces. This is why prompt optimization matters — the
 fewer tokens you use, the less you pay."

[Stage 2 — BPE Encoding]

"This is Byte-Pair Encoding — the actual algorithm GPT uses to turn text
 into tokens. Watch it step by step: it starts with individual characters,
 finds the most common pairs, merges them, and repeats. This is not a
 simulation — this is how it actually works."

[Stage 3 — Cost Calculator]

"Now here's the business case. Same prompt, four different models. Look at
 the cost difference."

[Point to the price comparison]

"GPT-4 costs 30x more than GPT-3.5 for the same input. If you're making
 10,000 API calls a day, that's the difference between a $50 monthly bill
 and a $1,500 monthly bill. Choosing the right model for the right task
 is not optional — it's survival."

[Stage 4 — Attention Map]

"This visualizes how tokens relate to each other. The AI doesn't read
 left to right like we do — it looks at ALL tokens simultaneously and
 measures how much each one should 'pay attention' to every other one.
 That's the transformer architecture in action."

[Stage 5 — Generation]

"And finally, generation. Watch the AI produce output one token at a time.
 Toggle between DETERMINISTIC mode — where it always picks the most
 likely next token — and CREATIVE mode, where it introduces randomness.
 That's what the 'temperature' slider does in every AI API."

KEY TALKING POINTS IF ASKED:
  - Tokenization is regex-based (approximation, not tiktoken — noted as future work)
  - BPE visualization is step-by-step, not just final result
  - Cost calculator uses real pricing from OpenAI, Anthropic, Google
  - All interactive — user types their own text

TRANSITION: "Now you understand HOW AI works. Let's BUILD something." → Click "Builder"

================================================================================
APPENDIX: TOUGH QUESTIONS & ANSWERS
================================================================================

Q: "Why not just use ChatGPT to learn prompting?"
A: "ChatGPT doesn't teach you tokenization costs, model differences, or
    structured prompt design. It's like learning coding by only using
    a compiler — you need a textbook + a lab. We're both."

Q: "The API is mocked — is this really useful?"
A: "The mock API demonstrates the full UX flow. Real API integration is
    one environment variable away — we built the architecture for it.
    The learning modules, builder, and patterns work independently of
    the API and provide immediate value."

Q: "Who is this for?"
A: "Three audiences: (1) Developers integrating AI APIs into products.
    (2) Non-technical users who want to get better results from ChatGPT.
    (3) Teams who need to standardize how they write prompts."

Q: "How is this different from PromptPerfect / PromptBase?"
A: "PromptPerfect optimizes prompts but doesn't teach you. PromptBase
    sells prompts but doesn't help you build your own. We do both:
    education + tooling, for free, in the browser."

Q: "What's the business model?"
A: "Open source educational tool. Future monetization through premium
    learning paths, team collaboration features, and hosted API proxy
    with cost analytics."
